## Directory Purposes

**`app/`:**
- Purpose: Next.js App Router — pages, layouts, API routes
- Contains: React components (pages), route handlers, global CSS
- Key files: `layout.tsx`, `page.tsx`, `api/count/route.ts`
- Convention: Page files use `"use client"` directive; API routes export named HTTP method functions

**`components/`:**
- Purpose: Reusable React components
- Contains: Feature components and shadcn/ui primitives
- Key files: `image-picker.tsx`, `result-display.tsx`, `annotated-image.tsx`
- Convention: All components use `"use client"` directive; kebab-case filenames

**`components/ui/`:**
- Purpose: shadcn/ui base components
- Contains: Pre-built UI primitives (button, card, badge)
- Key files: `button.tsx`, `card.tsx`, `badge.tsx`
- Convention: Generated by `npx shadcn@latest add <component>`; do not edit manually

**`lib/`:**
- Purpose: Business logic, utilities, external integrations
- Contains: CV pipeline, AI client, image processing, types, utilities
- Key files: `pill-cv.ts`, `pill-cv-config.ts`, `opencv-init.ts`, `openrouter.ts`, `pill-common.ts`, `compress.ts`, `utils.ts`
- Convention: Pure functions, no React dependencies (except `compress.ts` which uses browser APIs)

**`lib/opencv/`:**
- Purpose: Legacy/duplicate OpenCV initialization (unused)
- Contains: Duplicate config, init, and types files
- Key files: All are duplicates of files in `lib/` root
- Status: Can be safely deleted

**`docs/`:**
- Purpose: Project documentation
- Contains: Specs, tech stack reference, data flow diagrams
- Key files: `specs.md`, `tech-stack.md`, `count_flow.md`

**`public/`:**
- Purpose: Static assets served at `/`
- Contains: Default Next.js SVG icons
- Key files: None project-specific (all defaults)

**`test-img/`:**
- Purpose: Test images for development and debugging
- Contains: Sample pill images
- Key files: Test images for manual testing

## Naming Conventions

**Files:**
- Kebab-case for all files: `pill-cv.ts`, `image-picker.tsx`, `opencv-init.ts`
- `.tsx` for React components, `.ts` for pure logic
- Route handlers: `route.ts` inside route directory

**Directories:**
- Kebab-case: `components/ui/`, `lib/opencv/`
- Next.js conventions: `app/api/count/` for API routes

**Components:**
- PascalCase for component names: `ImagePicker`, `ResultDisplay`, `AnnotatedImage`
- Named exports (not default): `export function ImagePicker()`

**Functions:**
- camelCase: `countPillsWithCV`, `compressImage`, `buildPillCountPrompt`
- Descriptive verbs: `handleFile`, `handleCount`, `handleReset`

**Types/Interfaces:**
- PascalCase: `PillCountResult`, `PillCVConfig`, `OpenCVRuntime`
- Prefix with domain: `Pill*` for pill-related types

**Constants:**
- UPPER_SNAKE_CASE for config defaults: `DEFAULT_PILL_CV_CONFIG`
- camelCase for runtime constants: `OPENROUTER_MODEL`

## Where to Add New Code

**New UI Component:**
- Implementation: `components/<component-name>.tsx`
- If shadcn component: `npx shadcn@latest add <component>`
- Import from: `@/components/<component-name>`

**New API Endpoint:**
- Route handler: `app/api/<endpoint>/route.ts`
- Export named HTTP method: `export async function POST(request: NextRequest)`
- Set runtime if needed: `export const runtime = "nodejs"`

**New Utility/Helper:**
- Pure function: `lib/<utility-name>.ts`
- If related to CV: add to `lib/pill-cv.ts` or create `lib/pill-cv-<feature>.ts`
- If related to types: add to `lib/pill-common.ts`

**New Configuration:**
- CV pipeline config: `lib/pill-cv-config.ts`
- App config: `next.config.ts`
- Component config: `components.json`

**Test Images:**
- Place in: `test-img/`

## Path Aliases

- Configured in: `tsconfig.json`
- Usage: `import { cn } from "@/lib/utils"`, `import { Button } from "@/components/ui/button"`
- All imports use this alias (no relative `../../` paths)
