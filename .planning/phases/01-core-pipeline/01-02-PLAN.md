Implement `pill_counter/annotator.py` with a single function:

`annotate_image(image: np.ndarray, centers: list[tuple[int, int]], bounding_boxes: list[tuple[int, int, int, int]] | None = None) -> np.ndarray`

Per research code example (Annotator Implementation):
- Copy image (never modify original)
- For each center (1-indexed):
  - Draw filled red circle (BGR: 0,0,255) at centroid, radius 5
  - Draw sequence number text offset from center (+10, -10), red, font scale 0.7, thickness 2
- If bounding_boxes provided: draw green rectangles (0,255,0) around each pill
- Return annotated copy

Use cv2.putText with FONT_HERSHEY_SIMPLEX. Use cv2.circle with thickness=-1 for filled circle.

    - Test: corrupt image file is skipped, error logged, batch continues

  </behavior>
  <action>
Implement two modules:

**pill_counter/batch.py** — Batch orchestrator:
- `scan_images(folder: Path) -> list[Path]` — Find all JPG/PNG/WEBP/BMP files in folder (not recursive). Use pathlib.glob with case-insensitive matching (handle .JPG and .jpg). Return sorted list.
- `process_single_image(image_path: Path, output_dir: Path) -> dict` — Load image with cv2.imread, call count_pills(), call annotate_image(), save annotated to output_dir with same filename. Return dict with keys: filename, count, centers, status ('ok' or 'error'), error_msg (if error). Wrap entire function in try/except — on any exception, return dict with status='error' and error_msg=str(e). Per ERR-01: one bad image never kills the batch.
- `process_folder(input_dir: Path, output_dir: Path) -> list[dict]` — Call scan_images(), check empty (per ERR-02: raise ValueError with clear message if no images found). Create output_dir if not exists (per OUT-02). Loop over images calling process_single_image(). Return list of result dicts.

**pill_counter/cli.py** — argparse entry point:
- `main()` function with argparse:
  - Positional arg: `input` (folder path)
  - Optional: `-o/--output` (default: `./output`)
- Validate input path exists (exit 1 with error message if not)
- Call process_folder()
- Print terminal output: for each image, print `{filename}: {count} pills` (or `{filename}: ERROR - {error_msg}`)
- Print total: `Total: {sum of counts} pills across {n} images`
- Exit 0 on success, exit 1 on error (no valid images), exit 2 if some images failed

Add `[project.scripts]` in pyproject.toml: `pill-counter = "pill_counter.cli:main"` (if not done in 01-01).

Write tests FIRST: use tmp_path fixture to create test folders with synthetic images (use cv2.imwrite with circles), run process_folder(), verify output files exist and result dicts have correct counts. Test empty folder raises ValueError. Test corrupt file is skipped.
