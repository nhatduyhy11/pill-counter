Implement `pill_counter/pipeline.py` following the research document's Pattern 1 (Pure Function Pipeline) and Pattern 3 (Marker-Based Watershed).

**Functions to implement:**

1. `detect_background(gray: np.ndarray, sample_size: int = 20) -> str` — Per research Pattern 2: sample 4 corners (sample_size×sample_size pixels each), concatenate, return 'light' if median > 128 else 'dark'.

2. `auto_threshold(gray: np.ndarray, bg_type: str) -> np.ndarray` — Per research Pitfall 4: light background → THRESH_BINARY_INV + OTSU; dark background → THRESH_BINARY + OTSU.

3. `count_pills(image: np.ndarray, config: dict | None = None) -> PipelineResult` — Main pipeline function following research code example steps 1-13:
   - Grayscale conversion (handle both BGR and grayscale input)
   - Background detection via detect_background()
   - Gaussian blur with config blur_kernel_size
   - Threshold via auto_threshold()
   - Morphology: OPEN (remove noise) then CLOSE (fill holes) with config kernel/iterations
   - Distance transform, normalize to [0,1], threshold at config.distance_threshold × max
   - Sure background via dilate
   - Unknown = sure_bg - sure_fg
   - Connected components on sure_fg
   - CRITICAL per Pitfall 2: shift markers +1, set unknown to 0
   - Watershed on 3-channel image
   - Extract regions from watershed output (skip labels ≤ 1)
   - For each region: find contours, filter by min_area_ratio, max_area_ratio, min_circularity, min_solidity
   - Compute centroid via moments, bounding box via boundingRect
   - Return PipelineResult with count, centers, bounding_boxes, debug_images dict

**Critical implementation details from research:**
- MUST normalize distance transform before thresholding (Pitfall 3)
- MUST shift marker labels after connectedComponents (Pitfall 2: background=0 bug)
- MUST use median for corner sampling, not mean (Pitfall 6)
- Debug images dict should include: gray, binary, opened, closed, distance, sure_fg, markers
